**what currently exists?**
The project is a full-stack mobile application using Expo (React Native) for the frontend and FastAPI (Python) for the backend with a MongoDB database.

The application allows engineers to create detailed site visit reports through a multi-step form. Key functionalities include:
-   **Home Screen**: Lists all previously created site visit reports.
-   **Settings Screen**: Allows configuration of SMTP credentials for Gmail, and URLs for syncing Staff and Job lists from Google Sheets. It also includes a scheduled reporting feature.
-   **Report Creation Form**: A multi-step process:
    1.  **Site Info**: Select staff, job (with auto-fill for name/address), purpose of visit, site type, and other details. Includes address validation and a map link.
    2.  **Hazards**: Detail site risks and toolbox talks.
    3.  **Safety**: A comprehensive safety checklist with notes for each item.
    4.  **Inspection**: A detailed section for Building Consent Inspections, with conditional fields for different inspection types like Timber Pile Inspection.
    5.  **Photos**: Add photos from camera or library, with automatic naming and a comments field for each.
    6.  **Declare**: A final declaration with a signature pad, with the staff name auto-populated.
-   **Backend Services**:
    -   Generates a highly customized and branded PDF of the report.
    -   Sends the PDF report via email using configured SMTP settings.
    -   Provides an endpoint to email all photos from a specific report as attachments.
    -   Includes a scheduled task to email a summary spreadsheet of all reports at a configurable frequency.
    -   Provides a geocoding endpoint to validate addresses.
